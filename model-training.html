Synthetic → Real Object Detection

Unity Perception + SynthDet · YOLOv8 · Weights & Biases · Domain gap analysis

Synthetic Data YOLOv8 Training W&B Tracking Domain Shift Finetuning

Overview

Trained an object detection model starting from synthetic labeled images generated in Unity (Perception + SynthDet), then evaluated on real images and finetuned to reduce synthetic-to-real domain mismatch.

Pipeline

1) Synthetic dataset generation

  • Built a Unity scene + RGB camera prefab for image capture
  • Generated and exported labeled data using Unity’s SynthDet
  • Converted annotations to YOLO format and split train/val (8:2)

Unity Perception + SynthDet workflow and image capture are documented in the visuals PDF.

2) Training + logging

  • Trained YOLOv8 on synthetic images
  • Logged metrics + visualizations in Weights & Biases
  • Reviewed detection examples and confusion matrix to diagnose errors

Results

Synthetic-only training (63 classes)

Summary metrics reported after training on the synthetic dataset:

  • mAP@0.5: 0.248
  • mAP@0.5–0.95: 0.191
  • Precision: 0.287
  • Recall: 0.248

Key finding: synthetic → real gap

Before finetuning with real images, the model produced no correct detections across all classes on real photos, highlighting a strong domain shift (lighting, backgrounds, textures).

Finetuning with Real Images

To adapt the detector, I collected and labeled a real-image set and finetuned from the synthetic-trained checkpoint.

Real dataset

  • 604 real images (uploaded + labeled)
  • Converted to YOLO format; split train/val (8:2)
  • Used full-image bounding boxes (each image corresponds to one class)

Finetune metrics (real images)

  • mAP@0.5: ~0.252
  • mAP@0.5–0.95: ~0.249
  • Precision: ~0.289
  • Recall: ~0.264

Performance is constrained by dataset size and per-class coverage.

What I Learned

  • Domain shift is real: synthetic training alone didn’t transfer to real photos without finetuning.
  • Data quality dominates: limited real samples per class and coarse “full-image” boxes constrain learning.
  • Error analysis matters: confusion matrix + detection visualizations made failure modes obvious.

Next Steps

  • Collect more real images for underperforming classes (increase per-class diversity)
  • Use tighter, object-level bounding boxes instead of full-image boxes
  • Targeted finetuning / class rebalancing for low-recall classes
